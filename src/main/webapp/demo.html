A demo for showing major functions of GALLOC
(1). Log into the system with username and password. You can use a test user account which is already registered in the system with username as Tester and password as 123.
(2). Create a project by defining its name, geographic scope, category schema, annotator number, and batch size of messages. You will also need to upload data of social media messages to a project. We have provided a small example data containing 100 messages for testing in the GitHub repository at the link: https://github.com/geoai-lab/GALLOC. You can also play with the project "HurricaneHarvey" already created in the system.
(3). All ongoing projects are shown in the home page, where you can manage or play with each project.
(4). An annotation of location description is added by selecting its text in a message, choosing its category, and determining its spatial footprint.
(5). You can automatically pre-annotate a message using a pre-annotator. In the image, highlighted location descriptions are automatically recognized using the pre-annotator "Neurotpr". The URL of this pre-annotator is https://geoai.geog.buffalo.edu/preannotator/neurotpr.
(6). You can automatically identify the spatial footprint for a location description using map web services. In the image, the long spatial footprint for "Brays Bayou" is automatically identified using Nominatim.
(7). You can resolve potential disagreements among annotations from multiple annotators by accepting or revising based on one of them.
