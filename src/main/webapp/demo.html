A demo for using GALLOC, a GeoAnnotator for Labeling LOCation descriptions
from disaster-related text messages
This is a demo to demonstrate how GALLOC can be used for annotating location descriptions in social media messages. It involves the major components of GALLOC including system login, landing page, creating a new project, annotating a location description, automatic pre-annotation, automatic spatial footprint identification, and disagreement resolution.
System login. Log into the system with username and password. You can use a test user account which is already registered in the system with username as Tester and password as 123.
Landing page with projects. All ongoing projects are shown in the landing page, where you can manage or play with each project. We have created a project "HurricaneHarvey" with the test account.
Creating a new project. Create a project by defining its name, geographic scope, category schema, annotator number, and batch size of messages. You will also need to upload data of social media messages to a project. We have provided a small text corpus containing 100 messages for testing in the GitHub repository. You can also play with the project "HurricaneHarvey" already created in the system.
Annotating a location description. An annotation of location description is added by selecting its text in a message, choosing its category, and determining its spatial footprint.
Pre-annotating a message. You can automatically pre-annotate a message using a pre-annotator. In the image, highlighted location descriptions are automatically recognized using the pre-annotator "Neurotpr". The URL of this pre-annotator is https://geoai.geog.buffalo.edu/preannotator/neurotpr .
Identifying spatial footprint. You can automatically identify the spatial footprint for a location description using map web services. In the image, the long spatial footprint for "Brays Bayou" is automatically identified using Nominatim.
Resolving disagreements. You can resolve potential disagreements among annotations from multiple annotators by accepting or revising based on one of them.
For more details, please refer to the user manual.
